🎯 ¿Cuál es el propósito de la cartilla?

Esta cartilla digital tiene como propósito central explorar y analizar las heurísticas y funciones de evaluación como herramientas fundamentales en el desarrollo de sistemas de inteligencia artificial (Russell & Norvig, 2020). Busca proporcionar una comprensión profunda de cómo estos mecanismos permiten a los agentes inteligentes tomar decisiones eficientes en espacios de solución complejos.

Como señalan Tanenbaum y Bos (2024), "las heurísticas permiten encontrar soluciones aproximadas a problemas complejos de manera eficiente, cuando los métodos exactos serían computacionalmente prohibitivos" (p. 76).

Esta cartilla está dirigida principalmente a estudiantes de ingeniería y ciencias de la computación que se encuentran explorando el fascinante mundo de la inteligencia artificial. 🧠 Heurísticas y Funciones de Evaluación

🎯 Definición de Heurística

Una heurística es una técnica o estrategia que permite encontrar soluciones aproximadas a problemas complejos de manera eficiente, cuando los métodos exactos serían computacionalmente prohibitivos (Pearl, 1984).

🔍 Características Fundamentales

1. Eficiencia Computacional

Las heurísticas están diseñadas para reducir significativamente el espacio de búsqueda, permitiendo que los algoritmos encuentren soluciones en tiempo razonable.

Según Russell y Norvig (2020), "las heurísticas admisibles nunca sobreestiman el costo real para alcanzar el objetivo, garantizando soluciones óptimas en algoritmos como A*" (p. 95).

2. Aproximación Inteligente

Aunque no garantizan la solución óptima, las heurísticas bien diseñadas guían la búsqueda hacia regiones prometedoras del espacio de soluciones.

📊 Funciones de Evaluación

Las funciones de evaluación son mecanismos matemáticos que asignan un valor numérico a un estado específico, permitiendo comparar diferentes alternativas.

f(n) = g(n) + h(n) donde g(n) es el costo real y h(n) es la heurística

🎮 Aplicaciones Principales

  • Robótica: Planificación de rutas y navegación autónoma
  • Vehículos Autónomos: Toma de decisiones en tiempo real
  • Medicina: Diagnóstico asistido y planificación de tratamientos
  • Logística: Optimización de rutas y gestión de inventarios
Referencias: Hart, Nilsson y Raphael (1968) introdujeron el algoritmo A* que utiliza heurísticas admisibles para búsqueda óptima.

🎲 Teoría de Juegos en IA

🎯 ¿Qué es la Teoría de Juegos?

La teoría de juegos es un marco matemático para analizar situaciones donde múltiples agentes toman decisiones que afectan no solo su propio resultado, sino también el de otros participantes (Von Neumann & Morgenstern, 1944).

🤝 Elementos Fundamentales

👥 Jugadores

Agentes que toman decisiones.

🎯 Estrategias

Conjunto de acciones disponibles.

🏆 Pagos

Resultados según estrategias elegidas.

📊 Información

Lo que cada jugador conoce del estado.

🧠 Algoritmo Minimax

Tanenbaum y Bos (2024): "Minimax es fundamental en juegos de suma cero con información perfecta" (p. 142).
def minimax(estado, profundidad, es_maximizador): if profundidad == 0 or es_terminal(estado): return evaluar(estado) if es_maximizador: mejor_valor = -infinito for hijo in generar_hijos(estado): valor = minimax(hijo, profundidad-1, False) mejor_valor = max(mejor_valor, valor) return mejor_valor else: mejor_valor = infinito for hijo in generar_hijos(estado): valor = minimax(hijo, profundidad-1, True) mejor_valor = min(mejor_valor, valor) return mejor_valor

🔥 Ejercicio Práctico: Juego Matchstick

🎯 Objetivo del Ejercicio

Implementar y analizar un juego competitivo simple para inferir patrones de comportamiento y decisiones tomadas por un agente de IA mediante heurísticas y funciones de evaluación.

🎮 Reglas del Juego

  • Configuración inicial: Comienza con N palos de fósforo
  • Turnos alternados: Los jugadores retiran de 1 a 3 palos por turno
  • Condición de pérdida: El jugador que retira el último palo pierde
  • Estrategia: Forzar al oponente a retirar el último palo

💻 Algoritmo Minimax

Como explican Tanenbaum y Bos (2024), "el algoritmo Minimax es fundamental en juegos de suma cero con información perfecta, donde cada jugador asume que su oponente jugará de manera óptima" (p. 142).

def evaluar_estado(palos_restantes): """ Función heurística basada en el patrón módulo 4. Returns: 1 si favorable, -1 si desfavorable """ if palos_restantes % 4 == 0: return 1 # Posición ganadora else: return -1 # Posición perdedora

⚡ Algoritmo Minimax

def minimax(palos, turno_max, profundidad_max=10): """ Algoritmo Minimax para el juego Matchstick. palos: número de palos restantes turno_max: True si es el turno del maximizador profundidad_max: límite de búsqueda """ # Caso base: fin del juego if palos == 0: return -1 if turno_max else 1 # Caso base: límite de profundidad if profundidad_max == 0: return evaluar_estado(palos) mejores_puntajes = [] # Probar movimientos posibles (1, 2 o 3 palos) for movimiento in range(1, 4): if palos - movimiento >= 0: puntaje = minimax(palos - movimiento, not turno_max, profundidad_max - 1) mejores_puntajes.append(puntaje) # Retornar el mejor puntaje según el jugador if turno_max: return max(mejores_puntajes) else: return min(mejores_puntajes)

🎯 Función de Decisión de la IA

def obtener_mejor_movimiento(palos): """ Encuentra el mejor movimiento usando la heurística. """ remainder = palos % 4 if remainder == 0: # Posición perdedora, movimiento aleatorio return random.choice([1, 2, 3]) else: # Posición ganadora, llevar a múltiplo de 4 return remainder
